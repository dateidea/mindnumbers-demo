02
Upload all 23 images to Squarespace
One drag-and-drop into Custom Files gives you stable URLs
 ~5 min

Custom Files is where Squarespace stores the most stable asset URLs — exactly what we want for images the phone demo relies on.

  1. Log into Squarespace.
  2. In the left sidebar, go to Design → Custom CSS.
  3. Scroll to the bottom of the Custom CSS panel and click Manage Custom Files.
  4. Click Add Images or Fonts.
  5. Drag all 23 PNGs from Finder into the upload area. You can select them all at once with Cmd + A.
  6. Wait for uploads to finish. Keep this panel open — it'll be used again in Phase 04.
Why Custom Files? Image block URLs and gallery URLs can change when a page is re-published. Custom Files URLs stay put until the file is explicitly deleted — perfect for an embed that shouldn't break on its own.
03
Add the Code Block to your page
Paste the entire demo into one Squarespace Code Block
 ~1 min

One Code Block holds the entire interactive experience — layout, styles, and navigation logic in a single place.

  1. Go to the Squarespace page where the demo should live.
  2. Click Edit.
  3. Hover where the phone section should go, click the +, and choose Code from the block picker.
  4. In the Code Block editor:
    • Delete the default <p>Hello, World!</p> placeholder.
    • Toggle Display Source OFF (switch at the bottom of the editor) — Squarespace should render the HTML, not show it as text.
  5. Open squarespace-embed.html in any text editor.
  6. Select everything in the file (Cmd + A) and copy (Cmd + C).
  7. Paste into the Squarespace Code Block.
  8. Click Apply.
Expected at this point: the phone frame, hint cards, and layout all show up, but the screens inside the phone are blank. That's normal — Phase 04 fills them in.
04
Paste the 23 image URLs into MN_ASSETS
The single place in the code that maps each screen to its image
 ~10 min

At the bottom of the Code Block, inside the <script> section, there's a clearly fenced config object. This is the only place in the entire demo where image URLs live. Every screen is wired through it.

// =====================================================================
// EDIT HERE TO UPDATE IMAGES
// =====================================================================
var MN_ASSETS = {
  // --- SCREENS ---
  dashboardMain:       '',  // First screen you see when you open the app
  dashboardMyInput:    '',  // "My Input" tab of the dashboard
  moodDetails:         '',  // Detail page for the Mood & Speech card
  // ... 20 more lines, each with a comment telling you which image belongs there
};

How to fill it in

  1. Click the Code Block on your page and choose Edit.
  2. Open a second browser tab with Design → Custom CSS → Manage Custom Files (from Phase 02).
  3. For each of the 23 lines in MN_ASSETS:
    • Find the matching filename in Manage Custom Files (e.g. dashboard-main.png corresponds to the dashboardMain line).
    • Click the file — Squarespace automatically copies its URL to the clipboard.
    • Go back to the Code Block and paste the URL between the quotes: 
      dashboardMain: 'https://static1.squarespace.com/static/.../dashboard-main.png',
    • Move to the next line.
  4. After the first 2 or 3, click Apply and preview — if those screens render, the pattern is correct. Finish the rest.
  5. When all 23 are filled in, click Apply and Save the page.
Every line has a comment that tells you exactly which page of the app that image represents, so you can't mix them up.
05
Verify it works
Quick walkthrough to confirm every screen and interaction is wired up
 ~2 min

Preview the page (or view it published) and walk through each interaction.

Checklist

  • Phone shows the Dashboard screen with all 4 cards (Mood, Sleep, Activity, Routine).
  • Tapping a card on the phone navigates to that detail page.
  • Tapping the hint cards on the side (Sleep, Activity, Trends, Record, etc.) navigates to those screens.
  • The back arrow (top-left inside the phone) returns to the previous screen.
  • The bottom nav icons switch between Home, Trends, Record, Meds, More.
  • "Meds Taken" and "Your Prescription" tabs switch correctly inside the Meds screen.
If something is blank or broken: right-click the page → Inspect → open the Console tab. Any warning like Missing URL in MN_ASSETS for key: dashboardMyInput tells you exactly which line in the config is empty or has a bad URL. Fix that line and re-save.
06
Future workflow — swap any image in 30 seconds
Whenever the app redesigns, this is how the website stays current
 30 sec

When the app design changes and the demo on the site needs to reflect it, this is the full loop. No engineer required.

  1. Save the new PNG with the same filename as the old one (for example, a new Dashboard design gets saved as dashboard-main.png).
  2. Go to Squarespace → Design → Custom CSS → Manage Custom Files → Add Images or Fonts and upload the new PNG.
  3. Click the newly uploaded file in the Custom Files list — the URL is now on the clipboard.
  4. Edit the page with the demo, click the Code Block, and choose Edit.
  5. Scroll to the MN_ASSETS section.
  6. Find the line matching the page being updated (each line has a comment describing what page it is).
  7. Select the old URL between the quotes and paste the new URL over it.
  8. Click Apply and Save.

The page updates immediately for every visitor. That's the entire workflow.

One thing to know: Squarespace generates a brand-new URL every time a file is uploaded — even with the same filename. That's why pasting the new URL into MN_ASSETS is the one step that can't be skipped. It's a Squarespace platform behavior, not a design choice on our end.

Troubleshooting

Common gotchas
An image isn't showing up
Right-click the live page → Inspect → Console. Look for a warning like Missing URL in MN_ASSETS for key: dashboardMain — that's the exact line in MN_ASSETS that needs a URL. If the key already has a URL but the image still isn't loading, paste that URL into a new browser tab. If Squarespace returns a 404, the URL is wrong — re-copy it from Manage Custom Files.
Code block shows as raw text instead of rendering
The Display Source toggle at the bottom of the Code Block editor is turned ON. Turn it OFF — Squarespace will then render the HTML instead of showing it literally.
Hotspots and taps aren't responding
JavaScript in Code Blocks requires a Business plan or higher in Squarespace. Check the current plan under Settings → Billing. On Personal plans, Squarespace strips the <script> tag and the demo loses its interactivity.
Font looks off or layout is cramped
The demo loads the Inter font from Google Fonts. If Squarespace has global typography overrides, check Design → Site Styles → Fonts. The demo also has its own max-width (1100px) and padding — if the Squarespace section it lives in is unusually narrow, the layout may feel cramped. Placing the Code Block in a full-width section gives it the most room to breathe.
